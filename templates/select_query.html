Single Query

Takes one ID or entity name as input and finds all neighbors eminating from it.

Directions:


  1. Enter a single ID or entity name

  2. Specify depth: 1 for query's direct neighbors. >1 for recursion (Neighbor's neighbors, etc.).


Interactive Single Query Example - Senescence (mesh:D000375)

Multi Query

Takes a comma-separated list of IDs or entity names as input. Finds all possible pair combinations within this list. Finds either all of the simple paths between each pair that are shorter than a user-specified value, or all of the shortest paths between each pair.

Directions:


  1. Enter comma-separated list of IDs or entity names.

  2. Enter maximum number of edges between any pair of entities (<3 recommended).

  3. Select network type:
    • All simple paths
    • All shortest paths
